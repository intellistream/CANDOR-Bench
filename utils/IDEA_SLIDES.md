## Slide 1: Problem & Motivation

**标题**: Why Standard MVCC Doesn't Fit HNSW

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Background                                             │
│  • 实时 ANN 需要一致性 (并发读写)                       │
│  • HNSW 插入时会剪枝旧边 → 历史查询需要恢复             │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  T1: A ────── B    (边存在)                     │   │
│  │                                                  │   │
│  │  T2: Insert C, A-B 被剪掉                       │   │
│  │      A ──X─── B                                 │   │
│  │       \                                         │   │
│  │        ─── C                                    │   │
│  │                                                  │   │
│  │  T3: Query (snapshot = T1.5)                    │   │
│  │      需要恢复 A-B !                             │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  朴素方案: 存所有剪掉的边, 查询时全部恢复               │
│                                                         │
│  三个问题:                                              │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │                                                  │   │
│  │  ① 恢复效率低                                   │   │
│  │     大部分边无用, 但必须算距离才知道             │   │
│  │                                                  │   │
│  │  ② GC 不安全                                    │   │
│  │     删边可能破坏图连通性, 导致 recall 下降      │   │
│  │                                                  │   │
│  │  ③ 信息浪费                                     │   │
│  │     构建时已知的几何信息 (d_XD) 被丢弃          │   │
│  │                                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Our Idea: HNSW-MVCC Co-design                         │
│                                                         │
│    问题          →    解决方案                          │
│    ─────────────────────────────────────────────────   │
│    恢复效率低    →    存 d_XD, 几何过滤 (跳过99.7%)    │
│    GC 不安全    →    检查 in_degree, 保护连通性        │
│    信息浪费     →    存几何元数据, 查询时利用          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Slide 2: Module 1 - LF-MAOG (并发控制)

**标题**: Lock-Free Reads via Append-Only Property

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Observation: HNSW 节点 ID 单调递增, 边只增不改         │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Key Simplification: ID = Version                       │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │                                                  │   │
│  │  传统 MVCC:                                     │   │
│  │    Edge = {from, to, begin_ts, end_ts}          │   │
│  │    可见性: begin_ts ≤ snapshot < end_ts         │   │
│  │                                                  │   │
│  │  ANNchor:                                       │   │
│  │    Edge = {from, to}  // ID 本身就是版本!       │   │
│  │    可见性: max(from, to) ≤ snapshot             │   │
│  │                                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  示例:                                                  │
│  ┌────────────────────────────────────────────┐        │
│  │  Node 0 ── Node 1 ── Node 2 ── Node 3     │        │
│  │                                            │        │
│  │  Snapshot = 2:                            │        │
│  │    可见: Node 0, 1, 2  (id ≤ 2)           │        │
│  │    不可见: Node 3      (id > 2)           │        │
│  │                                            │        │
│  │  边 (1→3) 可见? max(1,3)=3 > 2, 不可见   │        │
│  └────────────────────────────────────────────┘        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Benefits:                                              │
│  • 省存储: 不需要 begin_ts, end_ts                     │
│  • 读无锁: append-only, 读取已有数据总是安全           │
│                                                         │
│  Result: 8 threads → 7.8x throughput                   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Slide 3: Module 2 - GAVC (版本存储)

**标题**: Store Geometry, Not Just Topology

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Observation: RNG 剪枝时几何信息已知, 不要丢!           │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  RNG 剪枝过程:                                          │
│                                                         │
│         X ──────────── D  (被剪掉)                     │
│          \    d_XD    /                                │
│           \  (已知!) /                                 │
│         d_XS\      /                                   │
│             \    /                                     │
│              \  /                                      │
│               S (dominator)                            │
│                                                         │
│  剪枝时我们知道:                                        │
│    • d(X, D) — 边的长度                                │
│    • S — 谁导致了剪枝 (dominator)                      │
│    • d(X, S) — 到 dominator 的距离                     │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  GAVC 存储:                                             │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │  struct GAVCEdge {                              │   │
│  │      int from, to;     // ID = 版本             │   │
│  │      float d_XD;       // 边长度 ← 关键!        │   │
│  │      int dominator;    // 谁剪的                │   │
│  │      float rp;         // 恢复潜力              │   │
│  │  };                                             │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  这些信息将用于下一个模块的高效过滤                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Slide 4: Module 3 - DILR (索引管理) ⭐核心

**标题**: Skip 99.7% Without Computing Distance

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  问题: 恢复历史边时, 大部分是无用的                     │
│        但不算距离不知道有没有用                         │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Key Insight: 利用存储的 d_XD, 不算距离就能判断!       │
│                                                         │
│            Q (query)                                    │
│            │                                            │
│     d_QX   │  (搜索过程中已知)                          │
│            │                                            │
│            X ─────────────── D (历史边)                │
│                   d_XD (GAVC 存储!)                    │
│                                                         │
│  三角不等式下界:                                        │
│                                                         │
│     d_QD ≥ √(d_QX² + d_XD²)                            │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │                                                  │   │
│  │  if √(d_QX² + d_XD²) > τ:                       │   │
│  │      skip(D)   // 不需要算 d_QD !               │   │
│  │                                                  │   │
│  │  高维空间: 下界 ≈ 真实值                        │   │
│  │  (HDCS: High-Dimensional Collinearity Sparsity) │   │
│  │                                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  朴素方案: 恢复 100% 边 → 算距离 → 80%+ 浪费           │
│  DILR:     几何过滤 → 跳过 99.7% → Recall 不变         │
│                                                         │
│  Result: 2.7x fewer distance computations              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## Slide 5: Module 4 - GAGC (垃圾回收)

**标题**: Graph-Aware GC: Protect Connectivity

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  问题: 删边可能破坏图连通性 → Recall 下降               │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  示例:                                                  │
│                                                         │
│  ┌─────────────────────────────────────────────┐       │
│  │     A ────── B ────── C                     │       │
│  │              │                               │       │
│  │              │ ← 唯一入边 (in_degree = 1)   │       │
│  │              ▼                               │       │
│  │              D                               │       │
│  │                                              │       │
│  │  朴素 GC: 过期就删 B→D                      │       │
│  │  结果: D 不可达, 搜索永远找不到 D           │       │
│  │  → Recall 下降!                             │       │
│  └─────────────────────────────────────────────┘       │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  GAGC 解决方案:                                         │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │                                                  │   │
│  │  if (过期                                       │   │
│  │      && in_degree[to] > 1    // 不是唯一入边   │   │
│  │      && rp < threshold):     // 低价值边       │   │
│  │      delete()                                   │   │
│  │  else:                                          │   │
│  │      keep()   // 保护连通性                    │   │
│  │                                                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  额外优化: 低 RP 的边优先回收 (反正很少被需要)          │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  Result: GC 后 Recall 保持不变                         │
└─────────────────────────────────────────────────────────┘
```

## Slide 6: How Modules Work Together

**标题**: Module Coordination

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  INSERT 流程                                            │
│  ────────────                                           │
│                                                         │
│  Vector v, id=5                                        │
│       │                                                 │
│       ▼                                                 │
│  ┌─────────┐                                           │
│  │ 找邻居  │ → 选中的边 → current graph               │
│  │ RNG剪枝 │ → 剪掉的边 → GAVC (存 d_XD, dominator)   │
│  └─────────┘                                           │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  QUERY 流程                                             │
│  ────────────                                           │
│                                                         │
│  Query Q, snapshot=1000                                │
│       │                                                 │
│       ▼                                                 │
│  ┌──────────────────────────────────────────────────┐  │
│  │  对每个访问的节点 X:                             │  │
│  │                                                   │  │
│  │  1. 遍历 current edges (只看 id ≤ 1000)         │  │
│  │                                                   │  │
│  │  2. 遍历 historical edges:                       │  │
│  │     ├─ 版本过滤: max(from,to) ≤ 1000 ?          │  │
│  │     ├─ HDCS过滤: √(d_QX² + d_XD²) > τ ? skip   │  │
│  │     └─ 通过 → 计算 d_QD                         │  │
│  │                                                   │  │
│  │  99.7% 历史边被跳过, 无需计算距离               │  │
│  └──────────────────────────────────────────────────┘  │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  GC 流程                                                │
│  ────────                                               │
│                                                         │
│  ┌──────────────────────────────────────────────────┐  │
│  │  遍历 historical edges:                          │  │
│  │    if 过期 && in_degree > 1 && rp < threshold:  │  │
│  │        安全删除                                  │  │
│  │    else:                                         │  │
│  │        保留 (保护连通性)                         │  │
│  └──────────────────────────────────────────────────┘  │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                     Summary                             │
│  ┌──────────┬─────────────────────┬────────────────┐   │
│  │ Module   │ HNSW特性利用        │ 效果           │   │
│  ├──────────┼─────────────────────┼────────────────┤   │
│  │ LF-MAOG  │ Append-only, ID递增 │ 读无锁,省存储  │   │
│  │ GAVC     │ RNG几何信息         │ 支持HDCS过滤   │   │
│  │ DILR     │ 三角不等式+高维特性 │ 99.7% skip    │   │
│  │ GAGC     │ 图连通性            │ Recall不降    │   │
│  └──────────┴─────────────────────┴────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 核心叙事总结

**Motivation (1句话)**:
> 朴素 MVCC 用于 HNSW 有三个问题：恢复效率低、GC 不安全、几何信息被浪费。

**Idea (1句话)**:
> HNSW-MVCC Co-design：针对 HNSW 特性重新设计 MVCC 各模块。

**Result (1句话)**:
> 99.7% 历史边跳过，Recall 不变，2.7x 更少距离计算。
