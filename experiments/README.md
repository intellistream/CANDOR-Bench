# GammaFresh Experiments

This directory holds the empirical work behind the GammaFresh paper.
**Start here.** All experiments are single-thread (`OMP_NUM_THREADS=1`)
to enable fair comparison across designs; multi-thread is acknowledged
as a known-honest limitation.

---

## Quick orientation

If you have 10 minutes, read **`FINDINGS.md`** — it's the synthesis,
top-level conclusions with numbers and tables.

If you have 30 minutes, also read **`PAPER_OUTLINE.md`** (paper structure
mapping each section to which experiment supplies its evidence) and
**`REVIEWER_CRITIQUE.md`** (hostile-but-fair peer review with proposed
defuses).

If you want to dig into a single experiment, each `eXX_*/` has its own
`README.md` (intent, command), `run.py` (script), and one or more
`output_*.json` (raw data).

## Experiment dirs (in narrative order)

### Foundation: characterize the workload landscape

| exp | what it answers |
|---|---|
| `e15_delete_pattern_matrix/` | Main 4-pattern × 2-backend × 2-scale matrix at SIFT 200K + 1M |
| `e16_cross_dataset/` | Same matrix on MSong, GloVe, random-m |
| `e17_churn_rate_sweep/` | Sensitivity to churn intensity (ratio 0.25× to 3.0×) |
| `e18_buffer_capacity/` | Sensitivity to buffer size (5× to 400× batch) |
| `e19_hnswlib_param_sensitivity/` | hnswlib (M, efC, efS) 13-config grid |

### Component-level ablation: why does it work?

| exp | what it answers |
|---|---|
| `e20_router_component_ablation/` | which simplified-router mechanism is load-bearing? (absorb-deletes is) |
| `e21_cpp_component_ablation/` | which native C++ machinery actually pays off? (mostly none at our scale) |
| `e22_cpp_tuned/` | can the C++ multi-partition design be saved by tuning? (no — partial_reset still broken) |
| `e23_multi_seed_variance/` | is the headline win statistically robust? (yes, ±1% over 5 seeds) |

### Modules added incrementally to the simplified Python POC

| exp | which module | does it help? |
|---|---|---|
| `e24_module_spatial_routing/` | K-partition + centroid routing | throughput-vs-latency tradeoff |
| `e25_module_tombstone_rebuild/` | periodic full rebuild on tombstone fraction | ★ -38% to -65% across patterns; THE big win |
| `e26_module_adaptive_maint/` | maint trigger on buffer-fill threshold | null result under qstride workload |
| `e27_module_cost_admit/` | per-batch admit decision via lifetime EMA | -28% on sequential; collapses on others |
| `e28_combined_modules/` | rebuild + cost-admit together | sequential best (combined -72%) |

### Reviewer-defense additions (post-peer-review)

| exp | what it answers |
|---|---|
| `e29_iso_rebuild_baseline/` | does the win come from the buffer or the rebuild? (rebuild dominates; buffer adds 5-15% on top) |
| `e30_workloads_rich/` | do rich modules (cost-admit, hot tier) help on workloads that need them? (mixed_lifetime + zipfian) |
| `e31_op_latency_decomposition/` | ★ where does the speedup come from? (insert 3000× faster + search 5× faster after rebuild) |
| `e32_iso_recall_pareto/` | does gamma+rebuild Pareto-dominate or just trade recall for time? (DOMINATES on SIFT 200K) |

### Pre-existing (e01-e14, before this sprint)

These produced the v1 → v3 thesis evolution; superseded by e15-e32 but
kept for reference. See `INTENTS.md` for one-paragraph summaries.

## Reproducing

```bash
# bootstrap (one-time)
bash setup_new_server.sh        # installs deps + downloads SIFT, MSong, GloVe

# any single experiment
OMP_NUM_THREADS=1 uv run python experiments/eXX_*/run.py

# all experiments in this sprint (a few hours single-thread)
for d in experiments/e[12][0-9]_* experiments/e3[0-2]_*; do
  OMP_NUM_THREADS=1 uv run python "$d/run.py"
done

# aggregate fresh tables + figures after running
uv run python experiments/_shared/aggregate.py
uv run python experiments/_shared/plot.py
```

## Shared infrastructure (`_shared/`)

| file | purpose |
|---|---|
| `data.py` | dataset loaders (SIFT, MSong, GloVe, random-m) + ground-truth + caching |
| `metrics.py` | recall@k, percentile latency |
| `workloads.py` | the standard streaming workload runner + 4 delete-pattern functions |
| `workloads_rich.py` | mixed_lifetime + zipfian_queries workload runners (for e30) |
| `workloads_instrumented.py` | per-op latency timing (for e31) |
| `backends.py` | base graph backends (HnswlibBackend, FaissHnswBackend) — both forced single-thread |
| `router.py` | the canonical simplified router: buffer + delete-absorb + lazy maint |
| `gamma_py_v3.py` | v2 with 4 component toggles (for e20) |
| `gamma_py_spatial.py` | K-partition spatial routing (for e24) |
| `router_with_rebuild.py` | tombstone-rebuild trigger (for e25) |
| `gamma_py_adaptive_maint.py` | buffer-fill auto-maint (for e26) |
| `gamma_py_cost_admit.py` | per-batch lifetime-EMA admit (for e27) |
| `gamma_py_combined.py` | rebuild + cost-admit combined (for e28) |
| `gamma_py_rich.py` | full rich version: per-vector life-table + heat-tracker + hot tier + γ-aware migration + per-partition rebuild (for e30) |
| `baseline_with_rebuild.py` | HnswlibDirectWithRebuild (for e29 — the missing baseline) |
| `builders.py` | configs for native C++ gamma index (for e21/e22) |
| `aggregate.py` | scan all `output_*.json` → ALL_RESULTS_TABLE.md + ALL_DELTAS_TABLE.md |
| `plot.py` | generate `figures/fig_*.png` from output JSONs |
